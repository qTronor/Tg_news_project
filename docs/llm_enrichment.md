# LLM Enrichment Service

## Overview

`llm_enricher` is an **optional add-on** microservice that generates natural-language explainability artifacts over pre-computed ML analytics. It does **not** replace base pipeline results (sentiment, NER, clustering, novelty scoring). All LLM-generated content is clearly flagged with `is_llm_generated: true` and a `disclaimer` field.

```
User / Frontend
      │  GET /analytics/clusters/{id}/llm/{type}
      ▼
Analytics API  ──(httpx POST /enrich/{type})──▶  llm_enricher  ──▶  Mistral API
      │                                                │
      │                                               ▼
      │                                         Postgres (llm_enrichments)
      │◀──────────────────────── cached JSON result ─────────────────────
```

## Architecture

| Component | File | Responsibility |
|---|---|---|
| `service.py` | `llm_enricher/llm_enricher/service.py` | aiohttp HTTP server, request routing |
| `handlers.py` | `llm_enricher/llm_enricher/handlers.py` | One `EnrichmentHandler` per enrichment type |
| `repository.py` | `llm_enricher/llm_enricher/repository.py` | `ClusterContextRepository` — aggregates input from 5 SQL queries |
| `cache.py` | `llm_enricher/llm_enricher/cache.py` | Postgres-backed TTL cache with `ON CONFLICT DO UPDATE` idempotency |
| `budget.py` | `llm_enricher/llm_enricher/budget.py` | In-memory daily cost tracker, warm-started from DB |
| `providers/mistral.py` | `llm_enricher/llm_enricher/providers/mistral.py` | Mistral API client with tenacity retry + pybreaker circuit breaker |
| `providers/mock.py` | `llm_enricher/llm_enricher/providers/mock.py` | Deterministic mock for dev/test |
| `prompts/` | `llm_enricher/llm_enricher/prompts/` | Versioned prompt templates per type × language |
| `fingerprint.py` | `llm_enricher/llm_enricher/fingerprint.py` | SHA-256 cache key derivation |

## Supported Enrichment Types (MVP)

| Type | Description | Output fields |
|---|---|---|
| `cluster_label` | Short label (≤80 chars) for a topic cluster | `{label, confidence}` |
| `cluster_summary` | 1-paragraph summary + key points | `{summary, key_points[]}` |
| `cluster_explanation` | Why this cluster is important (score drivers) | `{why_important, drivers[{name,weight,explanation}]}` |
| `novelty_explanation` | New / ongoing / resurgent verdict with rationale | `{novelty_verdict, rationale}` |

### Phase 2 (planned)
- `cluster_compare` — narrative comparison of two clusters
- `graph_explanation` — interpret top nodes and communities
- `question_answer_over_cluster` — free-form QA over cluster messages

## Language Support Matrix

| Language | `analysis_mode` | Prompt template | Output language |
|---|---|---|---|
| Russian (`ru`) | `full` | `prompts/<type>/ru/v1.txt` | Russian |
| English (`en`) | `full` | `prompts/<type>/en/v1.txt` | English |
| Other (any) | `partial` | `prompts/<type>/other/v1.txt` | English + partial-coverage note |
| Unknown (`und`) | `unknown` | `prompts/<type>/other/v1.txt` | English |

Language is determined by majority-vote over `raw_messages.language` for the cluster's messages.

## HTTP API

### `POST /enrich/{enrichment_type}`

**Request body:**
```json
{
  "public_cluster_id": "run1:0",
  "refresh": false
}
```

**Response:**
```json
{
  "status": "ok",
  "result": {"label": "Выборы 2024", "confidence": 0.91},
  "cached": true,
  "model": {"provider": "mistral", "name": "mistral-large-latest"},
  "prompt_version": "v1",
  "language": "ru",
  "analysis_mode": "full",
  "tokens": {"input": 1420, "output": 38},
  "cost_usd": 0.000515,
  "latency_ms": 1840,
  "generated_at": "2026-04-22T12:34:56Z",
  "is_llm_generated": true,
  "disclaimer": "This content was generated by an LLM and supplements (does not replace) base ML analytics.",
  "error": null
}
```

**Status values:** `ok | unsupported | budget_exhausted | error`

### Analytics API proxy routes

| Method | Path | Description |
|---|---|---|
| `GET` | `/analytics/clusters/{id}/llm/{type}` | Get cached result (202 if still computing) |
| `POST` | `/analytics/clusters/{id}/llm/{type}/refresh` | Force recomputation (30s timeout) |

## Caching

`cache_key = sha256(cluster_id | enrichment_type | language | prompt_version | model_name | input_fingerprint)`

`input_fingerprint = sha256(canonical_json(ClusterEnrichmentInput))`

- Default TTL: 7 days (`LLM_ENRICHER__CACHE__TTL_SECONDS`)
- Error TTL: 5 minutes (prevents retry storms)
- Invalidation: automatic when cluster data changes (fingerprint mismatch) or prompt version bumps
- Storage: `llm_enrichments` table (migration 012)

## Resilience

- **Retry**: tenacity, 3 attempts, exponential backoff (1s → 4s → 16s), on transport errors / 5xx / 429
- **Circuit breaker**: pybreaker, opens after 5 failures in 60s, half-open after 30s
- **Timeout**: 30s per Mistral call (`LLM_ENRICHER__LLM__TIMEOUT_SECONDS`)
- **Budget**: daily USD cap, returns `budget_exhausted` without calling Mistral

## How to Add a New Prompt Version

1. Create `llm_enricher/llm_enricher/prompts/<type>/<lang>/v2.txt` alongside the existing `v1.txt`.
2. The `PromptRegistry` automatically resolves "latest" to `v2`.
3. Old cache entries with `prompt_version=v1` naturally miss (different `cache_key`), triggering recomputation.
4. Do **not** edit existing `vN.txt` files — they are immutable.

## How to Swap Providers

1. Implement `LLMProvider` Protocol from `providers/base.py`.
2. Register in `service.py -> _build_provider()`.
3. Set `LLM_ENRICHER__LLM__PROVIDER=yourprovider`.

## Failure Modes

| Mode | HTTP status | Cached? | Retried? |
|---|---|---|---|
| Provider timeout | 500 (after retries) | Yes (5min TTL) | 3 times |
| Circuit breaker open | 500 | No | No |
| Budget exhausted | 200 (`status=budget_exhausted`) | No | No |
| Invalid JSON output | 500 | Yes (5min TTL) | Once |
| Unsupported enrichment type | 400 | No | No |
| LLM enricher down | 503 (from analytics_api) | — | No |
| Computing (timeout in analytics_api proxy) | 202 | — | Client polls |

## Future Extensions
